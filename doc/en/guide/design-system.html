Design System ​

Tbe UI provides a complete design system including colors, typography, spacing, borders and other design elements to help you build consistent user interfaces.

🎨 Design Principles ​

Tbe UI's design system follows these core principles:

  • Consistency: Unified design language and visual standards
  • Accessibility: Ensure all users can use comfortably
  • Flexibility: Support theme customization and personalized configuration
  • Performance Optimization: Lightweight design with excellent loading performance

🎯 Usage Guide ​

Color Usage ​

css
/* Using CSS variables */
.element {
  color: var(--t-color-primary);
  background-color: var(--t-bg-color);
  border-color: var(--t-border-color);
}

/* Using utilities */
<div class="text-primary">Primary text</div>
<div class="bg-primary">Primary background</div>

Typography Usage ​

css
/* Using CSS variables */
.title {
  font-size: var(--t-font-size-xl);
  font-weight: 600;

🚀 Theme Customization ​

CSS Variable Override ​

css
:root {
  /* Custom theme colors */
  --t-color-primary: #1890ff;
  --t-color-success: #52c41a;
  --t-color-warning: #faad14;
  --t-color-danger: #f5222d;
  
  /* Custom text colors */
  --t-text-color-primary: #262626;
  
  /* Custom background color */
  --t-bg-color: #fafafa;
}

Dark Mode ​

css
/* Dark mode */
@media (prefers-color-scheme: dark) {
  :root {
    --t-bg-color: #141414;
    --t-text-color-primary: #e8e8e8;
    --t-border-color: #434343;
  }
}

💡 Best Practices ​

  1. Consistency: Use the same design system throughout your application
  2. Hierarchy: Use appropriate font sizes and colors to distbeuish content importance
  3. Accessibility: Ensure color contrast meets WCAG standards
  4. Responsive Design: Use relative units and responsive utilities
  5. Performance: Prioritize CSS variables and utilities to reduce code duplication

By following Tbe UI's design system, you can create visually consistent and user-friendly interface designs.
